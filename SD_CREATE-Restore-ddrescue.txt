UGreen Model DXP4800 Plus
Copy ddrescue image(s) to the SD card.

Identify the SD card device

	lsblk -f
	
Assuming the SD card device is /dev/sdl

Here are two methods to copy the ddrescue image to the SD card.

Copy the entire NVMe image method:

	ddrescue -f backup/nvme2n1.img /dev/sdl sdl-restore.log

Copy each NVMe partition image method:

	ddrescue -f backup/nvme2n1p128.img /dev/sdl128
	ddrescue -f backup/nvme2n1p1.img   /dev/sdl1
	ddrescue -f backup/nvme2n1p2.img   /dev/sdl2
	ddrescue -f backup/nvme2n1p3.img   /dev/sdl3
	ddrescue -f backup/nvme2n1p4.img   /dev/sdl4
	ddrescue -f backup/nvme2n1p5.img   /dev/sdl5
	ddrescue -f backup/nvme2n1p6.img   /dev/sdl6
	ddrescue -f backup/nvme2n1p7.img   /dev/sdl7

If you copied a live file system when you made these image,
you will need to fsck to remove the dirty bit:

	fsck.vfat -va  /dev/sdl1
	fsck.ext4 -vfD /dev/sdl2
	fsck.ext4 -vfD /dev/sdl3
	fsck.ext4 -vfD /dev/sdl4
	fsck.ext4 -vfD /dev/sdl6
	fsck.ext4 -vfD /dev/sdl7

Don't touch the swap partition 5 nor the split-grub partition 128.
Do not try to mount nor check partition 128 without catastrophic results.
