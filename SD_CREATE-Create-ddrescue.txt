UGreen Model DXP4800 Plus
Create a backup of the boot NVMe

Identify the NVMe boot device.
Look for labels such as kernel, factory, UGREEN-SERVICE, and USER-DATA:

	lsblk -o NAME,FSTYPE,LABEL,UUID,PARTUUID,MOUNTPOINT
	
Assuming the NVMe device is /dev/nvme2n1

Here are two methods to copy the NVMe.

Copy the entire NVMe method:

	ddrescue -f -n /dev/nvme2n1 backup/nvme2n1.img backup/nvme2n1.map

Copy each partition method (recommended):

	ddrescue -f -n /dev/nvme2n1p128 backup/nvme2n1p128.img backup/nvme2n1p128.map
	ddrescue -f -n /dev/nvme2n1p1   backup/nvme2n1p1.img   backup/nvme2n1p1.map
	ddrescue -f -n /dev/nvme2n1p2   backup/nvme2n1p2.img   backup/nvme2n1p2.map
	ddrescue -f -n /dev/nvme2n1p3   backup/nvme2n1p3.img   backup/nvme2n1p3.map
	ddrescue -f -n /dev/nvme2n1p4   backup/nvme2n1p4.img   backup/nvme2n1p4.map
	ddrescue -f -n /dev/nvme2n1p5   backup/nvme2n1p5.img   backup/nvme2n1p5.map
	ddrescue -f -n /dev/nvme2n1p6   backup/nvme2n1p6.img   backup/nvme2n1p6.map
	ddrescue -f -n /dev/nvme2n1p7   backup/nvme2n1p7.img   backup/nvme2n1p7.map

Approximately 480G of data.

UGreens shipped kernel is here: https://github.com/ugreen-opensource

Also, since this is a copy of a live file system, there will be file corruptions.
All partitions that are active will be marked as dirty. After you restore
the data to the SD card, don't forget to run fsck to clean the file system.

To make a backup copy of the NVMe or SD card partition table:

	sfdisk -d /dev/nvme2n1 > nvme2n1_table.txt

If you ever need to restore the table:

	sfdisk /dev/nvme2n1 < nvme2n1_table.txt
